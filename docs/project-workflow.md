# Рабочий workflow проекта `ailit-agent`

Документ задаёт **обязательный** порядок работы агента и людей по этому репозиторию. Детальная стратегия этапов G–Q и далее — в [`../plan/ailit-global-agent-teams-strategy.md`](../plan/ailit-global-agent-teams-strategy.md). Краткий статус и входная точка — в [`../README.md`](../README.md).

## 1. Всегда вести работу по workflow в Markdown

1. Текущий нарезанный план задач — в `plan/` (в первую очередь `plan/ailit-global-agent-teams-strategy.md` и при необходимости `plan/agent-core-workflow.md`).
2. Перед началом итерации агент **открывает** соответствующий раздел плана и фиксирует, какие подзадачи закрываются.
3. После логического завершения подзадачи — **отдельный коммит** с понятным сообщением (этап/идентификатор задачи в префиксе или теле сообщения).
4. Проверки из плана (pytest, flake8, ручные сценарии CLI) выполняются **до** коммита, если план это требует.

## 2. Когда текущий workflow исчерпан

Если по документу workflow **нечего продолжать** (все пункты закрыты или нет утверждённой следующей ветки):

- агент **останавливается** и явно просит у человека **research** (обновление контекста, ссылки, новые референсы) и **постановку** следующей цели (новый фрагмент плана или отдельный workflow-файл);
- не придумывать объём нового этапа без согласования.
