# Workflow: hybrid memory и shared MCP-память (ailit-agent)

Документ задаёт **этапы → задачи → критерии приёмки** для внедрения паттерна **hybrid memory** (приватная рабочая память + shared KB / MCP) в экосистему `ailit`. Перед началом итерации агент открывает этот файл и фиксирует, какие задачи закрывает.

## Предусловие (порядок с первым workflow)

Этот файл ведётся **вторым** в цепочке. Сначала выполняется постановка и этапы **TE-0 … TE-5** в [`workflow-token-economy-recipe.md`](workflow-token-economy-recipe.md); переход сюда — только по **критерию перехода** из того документа (история решений, канонические ограничения, ответы или DEFERRED по TE-4, матрица TE-5).

Канонический порядок для людей и агентов также указан в корневом [`README.md`](../README.md).

Связанный рецепт, история решений и ограничения: [`workflow-token-economy-recipe.md`](workflow-token-economy-recipe.md).

Следующая итерация после закрытия этого файла: [`workflow-memory-3.md`](workflow-memory-3.md).

## Этап H0.0. Старт текущей итерации (kickoff)

Этот этап выполняется **первым при открытии** этого workflow в работе, до глубокого проектирования **H0.1**. Нужен, чтобы не начинать код «с середины» и чтобы handoff из [`workflow-token-economy-recipe.md`](workflow-token-economy-recipe.md) был явно проверен.

### Задача H0.0.1 — Верификация предусловия и handoff

**Промпт исполнителю:**  
«Открой [`workflow-token-economy-recipe.md`](workflow-token-economy-recipe.md), раздел критерия перехода и матрицу handoff. Зафиксируй в комментарии к PR или в теле коммита: какие пункты TE-* уже закрыты, какие помечены DEFERRED, какие ответы TE-4 перенесены сюда. Если handoff неполный — не начинай H0.1, вернись к recipe.»

**Критерии приёмки:**

- Есть явная запись (коммит/PR/issue): **handoff принят** или **handoff отклонён с причиной**;
- По каждому пункту из «Открытых вопросов» в конце этого файла: **ответ**, **DEFERRED** с датой и владельцем, или ссылка на строку в recipe, где это уже закрыто;
- Нет противоречий с каноническими ограничениями из recipe (scopes, запрет raw dump, index-first, конфликты).

**Тесты / проверки:**

- ручная сверка двух файлов side-by-side;
- при отклонении handoff — ноль строк нового кода в `ailit` под hybrid memory до повторного прохода recipe.

### Задача H0.0.2 — Рабочий лог спринта и выбор «первого ножа»

**Промпт исполнителю:**  
«Сформируй короткий план на 1–2 недели (или один milestone): какие задачи H0–H4 закрываешь в этой итерации и в каком порядке. Один primary deliverable: либо черновик контракта scopes (в сторону H0.1), либо карта MCP tools (в сторону H1.1) — выбери одно и обоснуй.»

**Критерии приёмки:**

- В репозитории есть одна запись плана спринта: issue, раздел внизу этого файла в блоке «Журнал итерации», или отдельный короткий doc в `plan/` со ссылкой отсюда;
- Выбран ровно **один** primary deliverable на старт;
- Указано, что сознательно **отложено** до следующей итерации.

**Тесты / проверки:**

- ручная проверка, что scope спринта не пересекает [`workflow-memory-3.md`](workflow-memory-3.md) (governance / multi-layer memory) без явного решения «вынести в M3».

**Тесты / проверки:**

1. Один MCP на организацию или отдельный экземпляр на workspace.
