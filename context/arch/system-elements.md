# Процессы ОС и верхнеуровневые элементы

Согласовано с инвариантом «один долгоживущий процесс ОС — один архитектурный элемент» (см. правило `architecture-os-process-invariant.mdc`). Краткосрочные вызовы CLI — одно PID-пространство на инвокацию.

| ID | Элемент | Точка входа / триггер | Ключевые пути |
|----|---------|----------------------|---------------|
| P1 | **CLI `ailit`** (короткоживущие команды) | `pyproject.toml` → `ailit_cli.cli:main` → `ailit/ailit_cli/cli.py` | Подкоманды регистрируются из `register_*_parser` (`runtime_cli`, `project_cli`, `desktop_cli`, …) и встроенных парсеров (`doctor`, `session`, `agent`, `memory`, …). |
| P2 | **Runtime supervisor** | `ailit runtime supervisor` (`ailit/ailit_cli/runtime_cli.py` → `run_supervisor_server`) | Долгоживущий; user unit из `scripts/install`. |
| P3 | **Broker и worker-процессы** | Под P2: `ailit runtime broker` (internal), дочерние процессы агентов из `ailit/ailit_runtime/` | Отдельные PID; broker Unix socket (отдельно от `supervisor.sock` — см. [`../proto/supervisor-json-socket.md`](../proto/supervisor-json-socket.md)). UC 2.4 (Work → Memory pathless, пост-guards в `memory_agent`): trace, инжект и W14-ветки — [`../proto/broker-memory-work-inject.md`](../proto/broker-memory-work-inject.md); канон envelope планера и журнал **`memory.command.normalized`** — `agent_memory_runtime_contract.py` / `agent_memory_query_pipeline.py`. **`propose_links`:** валидация кандидатов `AgentMemoryLinkCandidateValidator`, запись графа — `pag_graph_write_service.py` (`PagGraphWriteService`). Внешний конверт **`agent_memory.external_event.v1`** и типы событий (link_candidates, links_updated, …) — `agent_memory_external_events.py` / журнал **`memory.external_event`** из `memory_agent.py`. Журнал JSONL: поле **`durability`** (`memory_journal.py`) — `ephemeral` строки (напр. `memory.index.partial`) не пишутся на диск; durable шаги W14 и внешние финальные результаты остаются в trace/journal по правилам модуля. Компактные коды частичных исходов **OR-013** — `agent_memory_terminal_outcomes.py` (см. [`../algorithms/agent-memory/failure-retry-observability.md`](../algorithms/agent-memory/failure-retry-observability.md)). **UC-05:** cooperative cancel активного user-turn — broker JSON с **`runtime.cancel_active_turn`** (`broker.py` → AgentWork), поля корреляции **`chat_id`**, **`user_turn_id`** в payload; не supervisor JSON; Desktop — [`../proto/desktop-electron-runtime-bridge.md`](../proto/desktop-electron-runtime-bridge.md) §Cooperative Stop. Контракт SoT **`payload.agent_memory_result`**, **`memory_continuation_required`**, continuation gate в Work; таймаут RPC AW→AM — одна линия с `memory.runtime.agent_memory_rpc_timeout_s` (клиент Work, `svc_timeout` для worker AM в broker). D-OBS-1 — [`../proto/runtime-event-contract.md`](../proto/runtime-event-contract.md), `context/artifacts/architecture.md` §5. |
| P4 | **Desktop Electron** | `desktop/package.json` `main`; dev: `npm run dev`; prod: AppImage | `desktop/src/main/`, `desktop/src/renderer/`; UI — Chromium renderer. |
| P5 | **Удалённые UI** | Ранее: Textual TUI и Streamlit chat | Функциональность `ailit tui`, `ailit chat` и Streamlit `memory` без подкоманды удалена из репозитория; остаются CLI `ailit agent run`, `ailit memory <subcommand>`, desktop. |

## Исходящие связи

- P4 → P2/P3: IPC из main-процесса к `supervisor.sock` и broker socket; вызовы `ailit` для PAG slice (см. proto).
- P1 → P2: клиентские команды `ailit runtime status|brokers|stop-broker` и т.д.
- Тесты: `tests/conftest.py` изолируют пути `AILIT_*` от пользовательского `~/.ailit`. **UC-05:** инвариант порядка compact-событий после cooperative cancel — `tests/test_g14r_uc05_cooperative_cancel_trace_ordering.py` (hold: `AILIT_TEST_MEMORY_PIPELINE_HOLD_S`); Vitest на envelope — `desktop/src/renderer/runtime/envelopeFactory.cancel.test.ts` (см. [`../tests/INDEX.md`](../tests/INDEX.md)).

Доменные детали PAG / Memory 3D / W14 — в [`desktop-pag-graph-snapshot.md`](desktop-pag-graph-snapshot.md) и [`w14-graph-highlight-m1.md`](w14-graph-highlight-m1.md); этот файл — каркас learn по процессам.
