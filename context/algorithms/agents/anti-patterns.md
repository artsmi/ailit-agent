# Анти-паттерны реализации (Do not implement this as)

> **Аннотация:** список запретов, привязанных к **гейтам**, **ролям** и **путям артефактов**. Каждый пункт — **проверяемое** нарушение с последствиями для **01**, **105**, **11** или канона.

## Status

`draft`

## Связь с исходной постановкой

**OR-004** требует killer features **и** анти-паттерны «это не нужно / так делать нельзя». **OR-003** косвенно требует не смешивать идеи доноров с контрактом ailit. Синтез дополнительно требует фиксировать запреты: смешение **`plan/`** и плана **06**, подмена **09** через **11**, запись в **`context/**`** не через **13** после **12**, копирование donor prompts/skills.

## Общий принцип

**Слой для человека:** если реализация «упрощает жизнь» одной роли ценой подмены оркестратора или SoT, она **blocked** для этого репозитория. Анти-паттерн — не стиль кодирования, а **нарушение процесса**, которое в монорепо распространяется быстрее, чем review.

## Развёрнутые правила (failure shape)

### AP-1 / планы

**Forbidden:** сохранять выход **06** в файл под **`plan/17-*.md`** без явного решения человека и роли **106**, если это смешивает **wave-artifacts** с **implementation plan** target-doc.

**Allowed default:** **06** пишет план в каталог артефактов; **`plan/`** — отдельный контур **106/107** после approval.

### AP-2 / **11** и **09**

**Required:** **01** по **status** и completion gate определяет итог; **11** поставляет отчёт и **`stage_status`**, **не** «финальный merge gate» без **09** в волнах, где **09** обязателен.

### AP-3 / **`context/**`**

**Required:** изменения канона после разработки — **12** затем **13** в согласованных ветвях; **forbidden** прямой commit канона от **08**.

## Acceptance criteria

- Не менее **10** строк таблицы **AP** с явным гейтом или decision id.
- Каждый **AP** имеет **forbidden** действие и **последствие**.
- Нет ссылок читателю на временные артефакты сессии под **01** / черновики target-doc pipeline как на источник истины (**CR4**).
