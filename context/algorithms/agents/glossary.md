# Глоссарий пакета «Agents pipeline»

> **Аннотация:** термины ролей **02–09** и **11–13** под **01**, оркестрации **01** / **100**, путей планов и контуров **start-***. Текст самодостаточен для читателя канона: не требует открытия артефактов исследовательского пайплайна.

## Связь с исходной постановкой

**OR-005** предполагает, что читатель пакета может быстро разрешить коллизии имён: что такое «план» в корне репозитория и чем он отличается от выхода роли **06**, где живёт **исследование для канона** (только контур **`start-research`** и роли **101–108** под **100**), и почему **100** не заменяет **01**. Этот глоссарий фиксирует такие различия нормативно, чтобы постановки задач и ревью не смешивали контуры.

## Термины и сокращения

### Роли и оркестрация

| Термин | Определение |
|--------|-------------|
| **01 / orchestrator** | Роль **`01_orchestrator`**: владеет маршрутизацией feature/fix/learn по ролям **02–09** и **11–13**, гейтами завершения и записью прогресса; **единственная** роль, которая в контуре feature/fix/learn **запускает** Cursor Subagents для этих ролей (согласно правилам репозитория). |
| **02–09** | Последовательность постановки и проектирования до разработки: аналитик, ревью ТЗ, архитектор, ревью архитектуры, планировщик, ревью плана, разработчик, ревью кода. Каждая роль возвращает **JSON-first** ответ и согласованный markdown-артефакт; **не** инициирует вложенных субагентов. |
| **08 / developer** | Роль реализации кода по утверждённому плану; вход в типичные волны **`08 → 09 → 11`**. |
| **09 / code_reviewer** | Роль ревью кода после **08**; **passed** у **09** **не** заменяет ни завершение всего пайплайна, ни решение **01** о статусе сессии. |
| **11 / test_runner** | Независимая проверка (включая **pytest** и согласованные статические проверки); финальный **11** после волн разработки. |
| **12 / change_inventory** | Инвентаризация фактических изменений перед обновлением канона. |
| **13 / tech_writer** | Обновление согласованных фрагментов **`context/**`** по результату **12**; **не** пишет код продукта. |
| **100 / target_doc orchestrator** | Роль **`100_target_doc_orchestrator`**: владеет контуром утверждения целевых алгоритмов (**101–108**); вместе с **01** — единственный класс ролей, которому разрешён запуск Cursor Subagents в смысле правил репозитория. |
| **101–108** | Роли исследования доноров, текущего репо, синтеза, автора и верификатора target-doc, автора и ревьюера плана внедрения, reader review и оркестрации пакета human approval. |
| **Cursor Subagents** | Механизм IDE для запуска вложенных агентов; в репозитории привязан к **01** и **100**, не к **02+**. |

### Гейты, статус, артефакты

| Термин | Определение |
|--------|-------------|
| **JSON-first** | Ответ роли начинается с машиночитаемого JSON (например `stage_status`), затем markdown-артефакт или пояснение; оркестратор опирается на JSON на переходах. |
| **Gate / гейт** | Условие перехода, которое задаёт **01** (и описано в промпте оркестратора): например наличие файла постановки, прохождение ревью, лимит итераций. |
| **status / статус сессии** | Сводка прогресса, которую ведёт **01** в согласованном месте артефактов сессии; **не** поток типизированных событий как у stream-first фреймворков. |
| **SoT (source of truth)** | Для внедрения: утверждённый файл под `context/algorithms/…`, который **02** и далее обязаны читать, если задача относится к теме канона. |

### Пути «плана» (разводка обязательна)

| Термин | Определение |
|--------|-------------|
| **`plan/` (корень репозитория)** | Каталог **`plan/*.md`**: человекочитаемые планы внедрения, в том числе план для канона агентов (**`17-agents-pipeline-canon.md`** готовит **106** после утверждения target-doc), планы из target-doc workflow. **Не** автоматический выход одной роли **06** в изоляции. |
| **План артефактов роли 06** | Markdown-план, который производит **`06_planner`**, по договорённости репозитория располагается в **каталоге артефактов** текущего прогона оркестратора (wave-артефакты), а **не** обязан совпадать с файлами под **`plan/`** корня. Путать эти два смысла **запрещено**: иначе ломается трассировка задач и CI. |

### USER-story и маршрутизация

| Термин | Определение |
|--------|-------------|
| **USER-story** | Постановка от человека в терминах цели и ограничений; может требовать сначала утвердить target-doc (**start-research**), затем внедрение (**start-feature**), или идти сразу во fix/feature — см. [`user-story-handoff.md`](user-story-handoff.md). |
| **Только утверждение target-doc без кода** | Если менять продукт не требуется, а нужен утверждаемый алгоритм под `context/algorithms/`, маршрут — **start-research** / **100**, а не ведение target-doc внутри чата **01** как замена контура **101–108**. |
| **Исследование для канона** | Любое исследование доноров / текущего репо и синтез целевого алгоритма выполняется **только** в контуре **`start-research`** (**100** и **101–108**). Отдельной роли «исследователь у **01**» в нормативном пайплайне **нет**; устаревшее имя **`10_researcher`** в **`.cursor/`** снято (слайс **G17.3** плана внедрения). |

### Доноры и этика канона

| Термин | Определение |
|--------|-------------|
| **Donor / донор** | Внешний репозиторий-референс (**claude-code**, **opencode**, **ruflo**, **superpowers**): в канон переносятся **идеи** (цикл, события, навыки), **не** текст промптов и **не** исходный код. |
| **Provenance claude-code** | В human approval для target-doc фиксируется оговорка об ограничениях исходника; опубликованный канон опирается только на архитектурные выводы. |

### Проверяемость

| Термин | Определение |
|--------|-------------|
| **pytest из venv репозитория** | Запуск **`python -m pytest`** интерпретатором виртуального окружения проекта, чтобы субпроцессы видели те же зависимости. |
| **Изоляция тестов от домашнего каталога** | Автоподмена **HOME** и переменных **AILIT_*** для тестов, чтобы артефакты не писались в пользовательский **`~/.ailit`**; субпроцессы в e2e должны **наследовать** или **явно** получать те же переменные. |

## Известные зазоры (документировать, не скрывать)

| Зазор | Смысл |
|-------|--------|
| **`10_researcher` в `.cursor/`** | **Закрыто** в репозитории: строка и файл роли отсутствуют; исследование для канона только **`start-research`**. Исторические упоминания в **`plan/`** или в тексте канона — как ссылка на выполненный слайс **G17.3**. |
