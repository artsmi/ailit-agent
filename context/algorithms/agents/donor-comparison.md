# Сравнение с донорами (Donor comparison matrix)

> **Аннотация:** таблица измерений для поставки в крупных репозиториях: как **ailit-agent** расходится с **claude-code**, **opencode**, **ruflo** и **superpowers** по архитектурным решениям. Текст самодостаточен: пути к отчётам исследования читателю **не** требуются.

## Status

`draft`

## Связь с исходной постановкой

**OR-003** требует сопоставить доноров по циклу агента, инструментам, разрешениям, хукам, навыкам, планированию и масштабированию. Ниже каждое измерение даёт **одну строку поведения на систему**, чтобы **02** / **06** могли принять осознанное решение «что переносим в постановку» и «что не переносим», не открывая исходники доноров. Решения **migrate / do-not-migrate** — нормативные для репозитория **ailit-agent**, а не призыв копировать код.

## Why this exists

В монорепозитории ошибка «скопировать паттерн донора без контракта ailit» даёт смешение гейтов, дублирование оркестраторов и потерю проверяемости. Матрица фиксирует **наблюдаемые оси**, по которым доноры сильны или слабы относительно markdown-пайплайна **02–09** и **11–13** под **01**.

### Технический контракт (оси сравнения)

| Ось | Что сравниваем |
|-----|----------------|
| **SoT состояния** | Где лежит истина о прогрессе, контексте и результатах. |
| **Делегирование** | Кто создаёт субсессии / субагентов и кто владеет cleanup. |
| **События и наблюдаемость** | Типизированный поток vs JSON-гейты и файлы статуса. |
| **Владение артефактами** | Есть ли явная матрица «кто пишет файл X». |
| **Routing работ** | Фиксированные роли vs capability-router / skills graph. |
| **Compaction / контекст** | Явные слои урезания контекста vs политика пайплайна каноном. |
| **Hooks / bootstrap** | Инжект поведения до цикла (skills, session hooks). |
| **Swarm / параллелизм** | DAG задач, coordinator, message bus. |
| **Planning** | Централизованный план vs внутрисессионная координация. |
| **Канон и long-lived docs** | Отдельный слой алгоритмов в репозитории vs навыки/markdown только в сессии. |
| **Проверяемость workflow** | Как доказывается корректность цикла (тесты, grep, изоляция). |

## Матрица: доноры и ailit-agent

Легенда ячеек: краткая характеристика **по смыслу архитектуры**, без ссылок на конкретные файлы доноров.

| Ось | claude-code | opencode | ruflo | superpowers | ailit-agent |
|-----|-------------|----------|-------|-------------|-------------|
| **SoT состояния** | In-memory **State** + слои compaction | SQLite-сессии + типизированные события | Граф swarm / task state в шине | Skill + hook как стартовая политика | **артефакты прогона**, **`status.md`** у **01**, JSON ролей |
| **Делегирование** | `runAgent`, изолированный контекст, лимиты ходов | `task` tool, child session | Coordinator / DAG dispatch | Coordinator внутри сессии + SDD-сабагенты на задачу | **Только `01`** (feature/fix/learn) и **`100`** (target-doc) запускают Cursor Subagents |
| **События и наблюдаемость** | Пороги токенов, telemetry цикла | Discriminated union **SessionEvent** | Message bus, флаги компрессии | Hooks + тесты на stream-json skills | **`stage_status` в JSON**, отчёты ролей, **status** у **01** (не event-bus уровня SDK) |
| **Владение артефактами** | Центр: permissions; артефакты в рамках рантайма | Слабее явная owner-матрица ролей | Роли shell/swarm, не IDE markdown pipeline | Skills с приоритетом; reviews staged | **Жёсткая матрица 02–09 и 11–13** + **12→13** для **`context/**` канона |

| **Compaction / контекст** | **microcompact / autocompact / full** слои | Provider layer, reducer истории | План токенов vs факт реализации bus | Компактизация через skill-дисциплину | Сохранение канона в **`context/algorithms/**`**, обрезка «объёма» через этапы **02–07** |
| **Hooks / bootstrap** | В рамках продукта (не эквивалент markdown pipeline) | Конфигурация сессии / провайдеров | Entry CLI MCP | **SessionStart** hook инжектит skill | Вход через **`start-feature` / `start-fix` / `start-research`**, не hook в одной сессии |
| **Swarm / параллелизм** | Батчинг инструментов (чтение параллельно) | Делегирование в дочерние сессии | DAG + optional parallel | Параллельность через оркестрацию навыков | **Волны 08→09→11**; параллель **запрещён** для нарушения гейтов без **01** |

## Решения migrate / do-not-migrate (норматив для ailit)

### Слой для человека

Переносить из доноров стоит **идеи**, которые улучшают **наблюдаемость**, **безопасность мутаций** или **дисциплину контекста**, если они **встраиваются** в гейты **01** и не ломают запрет субагентов для **02+**. Переносить **нельзя** целые подсистемы (хуки одной сессии, arbitrary swarm router), если они подменяют оркестратора или **SoT** канона.

### Технический контракт

| ID | Паттерн донора | migrate (условно) | do-not-migrate (в этом репозитории как замена ролей **02–09** / **11–13**) |
|----|----------------|-------------------|----------------------------------------------------------|
| **M1** | Слои compaction (идея порогов) | Да, как **дисциплина этапов** и размера артефактов | Нет, как «модель сама жмёт всё без гейтов» |
| **M2** | Typed events / reducer | Да, как **форматы JSON** на выходе ролей | Нет, как обязательный глобальный event-bus без **01** |
| **M3** | Capability / semantic router swarm | Да, как **эвристика постановки задач человеком** | Нет, как автодиспетчер вместо **02–07** |
| **M4** | SessionStart skill bootstrap | Да, как **чеклист входа** в Cursor | Нет, как единственный **SoT** поведения вместо **`context/algorithms/`** |
| **M5** | Parallel tool read | Да, **внутри** роли, если не нарушает сериализацию мутаций по правилам продукта | Нет, как «параллельные субагенты от **08**» без **01** |
| **M6** | SDD fresh subagent per task | Да, как **анalogy** к волнам **08** | Нет, если это обход **`09`** / **`11`** |

## Как использовать при большом репозитории

1. Сначала открыть **`pipeline-topology.md`** и **`roles-and-artifacts.md`**: без них матрица доноров — справка, а не контракт.
2. Если команда предлагает «как в claude-code / opencode» — сверить ось **делегирование** и **владение артефактами**: в ailit **Subagents** только у **01** / **100**.
3. Если предлагают «swarm как ruflo» — сверить **routing**: capability-router **не** заменяет **06** и **09**.

## How start-feature / start-fix must use this

- **`02_analyst`** сверяет предлагаемый перенос паттерна донора с таблицей **migrate / do-not-migrate** и с **`killer-features.md`**.
- **`06_planner`** не трактует строки матрицы как задачи «внедрить донор целиком»; нарезка — в **`plan/*.md`** после approval канона.
- **`11_test_runner`** не использует этот файл как замену проверок **`pytest`**; доказательства цикла — в **`observability-and-tests.md`**.

## Acceptance criteria (пакет)

- Матрица покрывает оси **SoT**, **делегирование**, **события**, **владение**, **routing**, **compaction**, **hooks**, **swarm**, **planning**, **канон**, **проверяемость**.
- Нет требований читателю открывать временные markdown-артефакты target-doc pipeline или каталог артефактов сессии под **01** как обязательную опору (**CR4**).
- Есть блок **migrate / do-not-migrate** с проверяемыми запретами подмены **01**.

## Do not implement this as

- Не превращать матрицу в **очередь задач на копирование** репозиториев доноров.
- Не использовать матрицу как **permission** запускать Subagents из **02+** «потому что так удобнее».
