# Глоссарий пакета AgentMemory (Glossary)

Краткие расшифровки терминов, которые встречаются в каноне. Смысл фиксируется здесь, чтобы в остальных файлах можно было использовать сокращения после первого пояснения.

## Связь с исходной постановкой

Этот файл — **справочник сокращений** для всего пакета; он не подменяет нормативные разделы в `runtime-flow.md`, `llm-commands.md` и др., но задаёт единый словарь, без которого читатель не свяжет OR и технические термины между файлами.

| ID | Формулировка требования (суть, развёрнуто) |
|----|--------------------------------------------|
| **OR-001** | Канон AgentMemory разбит на файлы с `INDEX.md`; глоссарий обеспечивает **обнаруживаемость** смысла сокращений (OR, W14, SoT, D-OBS и т.д.) без дублирования длинных определений в каждом разделе. |
| **OR-DOC-001** | Утверждения блока «Текущая реализация» в пакете должны соответствовать коду; глоссарий помечает, что **`implementation_backlog`** — не ошибка канона, а явное расхождение «цель впереди кода». |
| **OR-DOC-002** | В doc-only workflow правки **только** в каноне и артефактах pipeline; глоссарий не требует product code, но термины в нём должны совпадать с wire-именами и enum в описаниях других файлов пакета. |
| **OR-DOC-003** | Нельзя выдумывать новую реализацию; расшифровки здесь отражают **уже зафиксированные** в каноне роли (например G13 как наследие, W14 как production-путь). |
| **OR-DOC-004** | Пакет `context/algorithms/agent-memory/` обновляется целиком; при добавлении нового сокращения в любой файл пакета его **нормативно** внести в таблицу ниже или явно расшифровать при первом вхождении в том файле, где термин появляется. |
| **OR-DOC-005** | Нарезка задач для кода живёт в `plan/` (вне канона); глоссарий не содержит слайсов `start-feature`, но термин **`implementation_backlog`** указывает читателю связь с планом внедрения по смыслу. |
| **OR-DOC-006** | При конфликте «целевое поведение» vs «текущая реализация» для **фактов сейчас** приоритет у текста «Текущая реализация» и модулей кода; глоссарий фиксирует семантику ярлыков (SoT, GAP), чтобы не перепутать норматив и факт. |
| **OR-DOC-007** | Корневой `context/algorithms/INDEX.md` может получать одну строку статуса; глоссарий **не** дублирует эту навигацию, но поддерживает согласованность терминов для ссылки из корневого индекса на пакет. |
| **OR-008** | Многоязычие и `file_kind` описаны в `prompts.md`; здесь — краткие якоря, чтобы читатель понимал, что **CoT** и **envelope** относятся к запретам и форме ответа LLM, а не к произвольному тексту модели. |
| **OR-009** | Каталог фаз промптов в `prompts.md`; глоссарий разводит **роль промпта** (инструкция к модели) и **wire-форму** ответа (`agent_memory_command_output.v1`), на которую ссылается `llm-commands.md`. |
| **OR-010** | Внешние события и компактные логи; в глоссарии **D-OBS** — подмножество наблюдаемости для брокера/потребителей, не исчерпывающее внутренний журнал. |
| **OR-011…OR-015** | CLI init, конверт результата, OR-013 (ошибки/повторы), сценарии и приёмка — в соответствующих файлах; глоссарий даёт общие метки (**GAP**, **TBD**, **Broker**, **PAG**), на которые опираются матрицы OR-013 и acceptance в `failure-retry-observability.md`. |

| Термин | Расшифровка | Комментарий |
|--------|-------------|-------------|
| **OR-00x** | Original Request — нумерованное требование из исходной постановки пользователя | В каждом файле пакета в разделе «Связь с исходной постановкой» перечислены только релевантные пункты OR с человекочитаемым текстом. |
| **W14** | Конвейер (pipeline) обработки запроса памяти в рантайме, опирающийся на строгий JSON-протокол ответа LLM | Якорные модули: `ailit/agent_memory/query/agent_memory_query_pipeline.py`, `ailit/ailit_runtime/subprocess_agents/memory_agent.py`. Не название продукта; в тексте для людей: «конвейер запроса памяти» или «протокол W14». |
| **G13** | Исторический планер памяти с походами A/B/C в свободном JSON до W14 | В текущем коде ответ на `memory.query_context` **не** строится через `AgentMemoryLLMLoop`; класс в `ailit/agent_memory/services/memory_llm.py` остаётся для тестов/наследия. |
| **SoT** | Source of Truth — единственный авторитетный источник для решения | Например: при конфликте полей продолжение сценария определяется по объекту `agent_memory_result`, а не по дублирующим полям выше по JSON. |
| **D-OBS** | Подмножество наблюдаемых событий, ориентированное на брокер и внешних потребителей | Не исчерпывает все записи внутреннего журнала памяти. |
| **GAP** | Пробел в документации или в проверке относительно целевого поведения | В таблицах handoff помечается типом `doc_incomplete`, `verification_gap` и т.д. |
| **TBD** | To Be Determined — решение намеренно отложено | Нужно явно указать, что именно ещё не выбрано. |
| **implementation_backlog** | Запланированное расхождение: целевое поведение описано в каноне, код ещё не выровнен | Не ошибка канона; напоминание для `start-feature` / `start-fix`. |
| **CoT** | Chain-of-thought — пошаговые рассуждения модели вне структурированного ответа | В машинном канале запрещены: ответ только в разрешённой JSON-схеме. |
| **Envelope command** | Верхнеуровневая команда в JSON-ответе LLM (поле `command`) | Отделяется от «действий планера» внутри `plan_traversal`. |
| **Broker** | Компонент, который принимает запросы от агентской оболочки и передаёт их в подпроцесс памяти | Транспорт: обмен строками JSON (JSON Lines) по stdin/stdout подпроцесса. |
| **PAG** | Граф памяти в SQLite (Persistent Agent Graph) | Узлы и рёбра с типами A/B/C/D. |
| **KB** | Knowledge base — вспомогательное хранилище знаний в контексте продукта | На пути обычного `memory.query_context` в worker может не использоваться; на пути init — по политике транзакций. |

При добавлении новых сокращений в пакет их нужно внести в эту таблицу или расшифровать при первом вхождении в том файле, где термин появляется.
