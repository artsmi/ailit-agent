# Глоссарий пакета AgentMemory (Glossary)

Краткие расшифровки терминов, которые встречаются в каноне. Смысл фиксируется здесь, чтобы в остальных файлах можно было использовать сокращения после первого пояснения.

| Термин | Расшифровка | Комментарий |
|--------|-------------|-------------|
| **OR-00x** | Original Request — нумерованное требование из исходной постановки пользователя | В каждом файле пакета в разделе «Связь с исходной постановкой» перечислены только релевантные пункты OR с человекочитаемым текстом. |
| **W14** | Конвейер (pipeline) обработки запроса памяти в рантайме, опирающийся на строгий JSON-протокол ответа LLM | Не название продукта; внутренний ярлык реализации. В тексте для людей: «конвейер запроса памяти» или «протокол W14». |
| **SoT** | Source of Truth — единственный авторитетный источник для решения | Например: при конфликте полей продолжение сценария определяется по объекту `agent_memory_result`, а не по дублирующим полям выше по JSON. |
| **D-OBS** | Подмножество наблюдаемых событий, ориентированное на брокер и внешних потребителей | Не исчерпывает все записи внутреннего журнала памяти. |
| **GAP** | Пробел в документации или в проверке относительно целевого поведения | В таблицах handoff помечается типом `doc_incomplete`, `verification_gap` и т.д. |
| **TBD** | To Be Determined — решение намеренно отложено | Нужно явно указать, что именно ещё не выбрано. |
| **implementation_backlog** | Запланированное расхождение: целевое поведение описано в каноне, код ещё не выровнен | Не ошибка канона; напоминание для `start-feature` / `start-fix`. |
| **CoT** | Chain-of-thought — пошаговые рассуждения модели вне структурированного ответа | В машинном канале запрещены: ответ только в разрешённой JSON-схеме. |
| **Envelope command** | Верхнеуровневая команда в JSON-ответе LLM (поле `command`) | Отделяется от «действий планера» внутри `plan_traversal`. |
| **Broker** | Компонент, который принимает запросы от агентской оболочки и передаёт их в подпроцесс памяти | Транспорт: обмен строками JSON (JSON Lines) по stdin/stdout подпроцесса. |
| **PAG** | Граф памяти в SQLite (Persistent Agent Graph) | Узлы и рёбра с типами A/B/C/D. |
| **KB** | Knowledge base — вспомогательное хранилище знаний в контексте продукта | На пути обычного `memory.query_context` в worker может не использоваться; на пути init — по политике транзакций. |

При добавлении новых сокращений в пакет их нужно внести в эту таблицу или расшифровать при первом вхождении в том файле, где термин появляется.
